## OPTIMIZATION PLAYBOOK — Scaling, Pausing, and Campaign Management

### Rule 1 — Never Change a Learning Ad Set

During learning phase (status = "LEARNING"):
  - Do NOT change budget by more than 20%
  - Do NOT change targeting
  - Do NOT change optimization goal
  - Do NOT pause for > 7 days
  Wait until: learning exits (SUCCESS or FAIL) before making changes.

If you must intervene during learning (budget almost gone):
  - Increase budget ≤20% at a time to avoid resetting
  - Or do nothing and let it fail, then rebuild with adjustments

---

### Rule 2 — Minimum Spend Before Pausing

Do NOT pause ad set based on:
  - 0 conversions with < $20 spend (not enough data)
  - High CPA with < 30 clicks (statistical noise)
  - First 3 days of running

DO pause ad set when:
  - Spend > $50 AND conversions = 0
  - Spend > 3x target_cpa AND conversions < 1
  - learning_status = FAIL and budget changes don't help
  - CTR < 0.3% with > 10k impressions (creative fatigue)

---

### Rule 3 — Budget Scaling Protocol

Criteria to scale (increase budget):
  - Ad set exited learning (status = SUCCESS)
  - CPA < 0.8x target_cpa
  - At least 10 conversions in the last 7 days
  - Spend has been stable for 3+ days

How to scale:
  - Step 1: Increase budget by exactly 20% (to avoid resetting learning)
  - Step 2: Wait 3 days minimum before next increase
  - Step 3: Repeat if CPA remains below target

Aggressive scaling (for strong performers):
  - If CPA < 0.5x target and volume > 20 conv/week:
    - Duplicate the ad set (fresh learning, new budget)
    - Keep original running while duplicate learns
  - If lookalike 1% hits 30+ conv: test lookalike 2-3% as new ad set

---

### Rule 4 — Creative Refresh Triggers

Time to refresh creative when:
  - Frequency > 3.0 AND CTR declining week-over-week
  - CTR drops below 50% of initial CTR
  - Ad has been running > 21 days to the same audience
  - Cost per click increasing > 30% week-over-week

How to refresh:
  - Do NOT edit existing ad (pauses learning)
  - Create a new ad within the same ad set with new creative
  - Run old and new ad simultaneously — Meta will optimize delivery
  - After 7 days: pause the underperforming ad

---

### Rule 5 — Audience Exhaustion

Signs of audience exhaustion:
  - Frequency > 4.0
  - CPM increasing > 30% week-over-week
  - Reach has peaked and stopped growing
  - CTR declining despite low frequency (audience quality issue)

Solutions:
  1. Expand age range (e.g. 25-44 → 22-50)
  2. Add 2-3 new interest categories
  3. Switch to broader lookalike (1% → 3%)
  4. Test new geographic markets
  5. Use Advantage+ audience (let Meta find similar users automatically)

---

### Optimization Decision Tree

```
Is learning_status = FAIL?
  YES → Increase budget or broaden targeting → re-launch
  NO ↓

Is learning_status = LEARNING?
  YES → Do nothing, wait for exit → check again in 3 days
  NO ↓ (status = SUCCESS)

Is CPA > 2x target?
  YES AND spend > $50 → PAUSE ad set
  YES AND spend < $50 → Wait for more data
  NO ↓

Is CPA between 0.8x and 2x target?
  YES → MAINTAIN (no changes needed)
  NO ↓ (CPA < 0.8x target)

Is conversion volume >= 10 last 7 days?
  YES → SCALE budget by +20%
  NO → Wait until volume increases
```

---

### Budget Reallocation After Pausing

When pausing an underperforming ad set, redistribute its budget:
  - 60% to best performing ad set
  - 40% to second best (or keep for future test)
  - Reduce budget changes incrementally to avoid resetting learning of recipients

Example:
  Before: Cold $40/day (pausing), Warm $30/day, Lookalike $30/day
  After: Warm $48/day (+20% of $40 = $8 extra), Lookalike $38/day (+$8 extra)
  Note: redistribute over 2-3 days if amounts are large

---

### A/B Testing Protocol

To properly A/B test creatives or audiences:
  - Create separate ad sets (not ads) for proper budget isolation
  - Run for minimum 7 days or until each side has 50+ events
  - Use identical settings except the variable being tested
  - Only change ONE variable at a time (creative OR audience, not both)
  - Statistical significance: winner should have CPA 20%+ better with 100+ events

---

### Campaign Type Reference

For mature campaigns (past test phase, proven CPA):
  Consider switching to:
  - ADVANTAGE+ SHOPPING CAMPAIGNS (for e-commerce) — Meta automates everything
    Creates one campaign, Meta handles targeting + bidding + creative optimization
    Requires: product catalog + pixel purchase event
  - CBO (Campaign Budget Optimization) instead of per-ad-set ABO:
    Set budget at campaign level, Meta distributes to best performing ad sets
    Activate: POST /{campaign_id} {"daily_budget": amount, "budget_rebalance_flag": true}

### Meta Ads Policy Quick Reference

Always check:
  - No misleading claims or before/after images (health, weight loss)
  - No targeting based on sensitive attributes (health, religion, political views)
  - Landing page must match ad content and be functional
  - Special ad categories: CREDIT, EMPLOYMENT, HOUSING, SOCIAL_ISSUES_ELECTIONS_OR_POLITICS
    require additional declaration in campaign creation
